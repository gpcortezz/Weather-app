3.1. Diseño General:

Arquitectura de aplicación web del lado del cliente (frontend) con enfoque en Progressive Web App. La aplicación se ejecuta completamente en el navegador y consume APIs externas para obtener datos meteorológicos.

3.2. Capas y Módulos:

  • Capa de Presentación: HTML templates y páginas (index.html, pages/*.html)
  • Capa de Lógica: Módulos JavaScript (scripts/app.js, scripts/api.js, scripts/forecast.js, scripts/location.js, scripts/menu.js)
  • Capa de Servicios: Service Workers para funcionalidad offline
  • Capa de Datos: APIs externas de servicios meteorológicos

3.3. Patrones de Diseño:

  • Módulos ES6 para separación de responsabilidades
  • Patrón MVC (Model-View-Controller) implícito en la estructura
  • Service Worker para patrón de caching y offline first

3.4. Flujo de Datos Principal:

  1. Usuario accede a la aplicación
  2. Se obtiene la ubicación del usuario mediante Geolocation API
  3. Se realiza petición a API meteorológica externa con las coordenadas
  4. Se procesan y muestran los datos en la interfaz
  5. Service Worker cachea recursos para funcionamiento offline
