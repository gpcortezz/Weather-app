3.1. Diseño General:

Aplicación web progresiva (PWA) con arquitectura cliente-servidor. El frontend se comunica con una API externa para obtener datos meteorológicos.

3.2. Capas y Módulos:

  • Presentación: HTML/CSS
  • Lógica de negocio: JavaScript (app.js, forecast.js, location.js)
  • Servicio: API externa para datos meteorológicos
  • Persistencia: Cache API para soporte offline

3.3. Patrones de Diseño:

  • Módulos ES6 para organización del código
  • Service Worker para caché y soporte offline
  • Singleton para el manejo del estado de la aplicación

3.4. Flujo de Datos Principal:

  1. El usuario accede a la aplicación
  2. Se obtiene la ubicación del dispositivo
  3. Se consulta la API meteorológica con las coordenadas
  4. Se muestran los datos procesados al usuario
  5. Los datos se almacenan en caché para uso offline
