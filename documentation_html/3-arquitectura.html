3.1. Diseño General:

Aplicación web progresiva (PWA) con arquitectura cliente-servidor. El frontend se comunica con una API externa para obtener datos meteorológicos.

3.2. Capas y Módulos:

  • Capa de Presentación: HTML/CSS
  • Capa de Lógica: JavaScript (app.js, forecast.js, location.js)
  • Capa de Servicios: API externa (api.js)
  • Capa de Soporte Offline: Service Worker (sw.js)

3.3. Patrones de Diseño:

  • Módulos ES6 para organización del código
  • Patrón Singleton para el Service Worker
  • Cache-first strategy para recursos estáticos

3.4. Flujo de Datos Principal:

  1. El usuario accede a la aplicación
  2. Se obtiene la ubicación mediante Geolocation API
  3. Se consulta la API meteorológica con las coordenadas
  4. Se muestran los datos procesados en la interfaz
  5. Los recursos se cachean para uso offline
